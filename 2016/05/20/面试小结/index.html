面试小结

面试 总结 2016-05-20

这次面试经历有些特别，两个小时面了三次
对面试过程中印象深刻的问题做下总结

在内推回绝后，经过了笔试，收到阿里的面试通知，与腾讯面试相同的酒店面试，让我多少感觉比较放松。
提前40分钟到达面试地点，报名登记后没多久就收到了一面通知。
一面前辈，问的几个问题让我觉得非常有趣，也感受到自身许多的不足,在此记录总结一下。

一面

在对写的小项目的询问后，进入知识考察环节（=。=姑且这么叫吧）

Q1.如合在canvas上画一条一像素宽的线/一个一像素的点

现在回忆了下，发现当时误会了考官的意思，我理解成画一条一像素的线，当时懵比的回答canvas能进行像素级的操作BALABALA..
其实考官是要考察，如何在canvas中画出奇数宽度的线，冏
如果直接向这样定义线的宽度

ctx.lineWidth = 1;
ctx.beginPath();
ctx.moveTo(100,100);
ctx.lineTo(100,150);
ctx.stroke();

是没有作用的画出的将是俩像素宽的线，原因可以参考html5-canvas-and-line-width

对奇数宽度的线宽使用小数处理

ctx.lineWidth = 1;
ctx.beginPath();    
ctx.moveTo(100.5,100);
ctx.lineTo(100.5,150);
ctx.stroke();

这样画出的线的颜色和宽度才会符合预期。

至于画出一个点，可以直接看这里drawing-a-dot-on-html5-canvas

Q2.[1,2,3].map(parseInt)会输出什么？

答案是[1, NaN, NaN];
map函数会给接受的参数传递value和index两个参数

parseInt() 函数将给定的字符串以指定基数（radix/base）解析成为整数(十进制的)。
parseInt会接受两个参数parseInt(string, radix);
string是需要转换的值，radix是数制的基数;
如果对应的基数的数值不存在则返回NaN

这道题在考官引导下也没有回答正确，还是要加强基础知识，对常用函数还应加深了解。

Q.3怎么实现小于12px的字体大小

这个需求在之前的开发过程中没有处理过，因为chrome规范要求不应存在这么小的字体，但实际需求是存在的

当时我只想到既然字体的限制是无法改变的只能对渲染结果做处理，那就是transform

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <style type="text/css">
        p:nth-of-type(1){
            transform: scale(0.5, 0.5);
        }        
    </style>
</head>
<body>
    <p style="font-size:6px;">test 6px</p>
    <p style="font-size:12px;">test 12px</p>
</body>
</html>

需要注意的是transform属性只能对具有block属性的元素起作用，因此对生效需要将其diplay设置为inline-block。并且变形也会导致bakgrond变形，因此应当将文字元素与背景元素分离。

Q.4 为什么inline-block的li之间有多余的像素，如何去除

这个问题当面试官问出后，勾起我许多回忆，在一年多前，我发现自己不论如何设置li的css属性，他们之间总是有间隔，后来我删除掉了li间的所有字符才解决的，当时觉得莫名奇妙。一切诡异的现象都有其合理的解释。

问题代码如下：

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <style type="text/css">
        li{
            display: inline-block;
            width: 100px;
            margin: 0px;
            padding: 0px;
            background-color: green;
        }
    </style>
</head>
<body>
    <ul>
        <li>1</li>
        <li>2</li>
        <li>3</li>
        <li>4</li>
        <li>5</li>
    </ul>
</body>
</html>

出现间隔的原因，因为li的display设置为inline-block,可以将li看作文字元素类似于span，span本身就是行内你元素，li标签之间存在一个换行字符，因此出现了空白。

解方法：CSS-TRICKS有一篇文章fighting-the-space-between-inline-block-elements

解决方法很巧妙，然而一切都应基于了解原理的前提下才能想出这些有趣的方法。

这个问题本身的价值或许不大，但让我突然意识到联系前后知识，以及对出现特殊状况应当保持足够用的敏感，也许就能从问题当中有所收获。

一面的问题就写到这里吧，如果还想起什么特别的再补充。

二面

二面主要侧重于项目，先是问了问学习的经历，然后根据简历上的项目进行提问。
有趣的跟面试官提起我写的小游戏坦克大战之前心血来潮，觉得很有趣就做的。并且提起了想最终实现联机功能。
这个项目模块化方面做的很原始，还是有些心虚的。

面试官就做过类似的项目，有幸得到关于项目细节的指点。

  1. 使用HTML5制作，视觉效果的问题大部分都能解决，但音乐效果是个大坑，HTML5的音乐API功能比较简陋，无法实现音轨的叠加，不能同时播放。（我自己之前只简单的添加了音效，当时觉得有些嘈杂，也没有详细去做，今后尝试下）

  2. 联机，联机将指令分为两类，操作指令和绘图指令，这样才能保障在丢包的情况下，始终保持每个客户端的状态一至。（天了噜，这个我在设计之初就想错了，计划只用操作指令同步的，因此绘图的接口没有预留，如果想要实现联机功能，估计要做大的改动）

  3. 绘制周期不一定要调用RAF，通过对游戏数据的计算，可以得到合适的刷新周期，这样能使帧数更稳定，体验更好。（这个问题比较复杂了。。）

问了关于自适应网站开发的项目，我就提了这个网站，但面试官打开的时候，布局出现了问题，我很方啊。。回学校路上赶忙试了试，发现是正常，有些郁闷。
