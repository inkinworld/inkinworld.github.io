inline-block VS inline

CSS 2016-07-24

这个算经典问题了，对布局挺有用
考完试就开始实习啦，日常要做项目，没太多时间写东西
借周末写一点

W3School上通常解释到:

inline-block 就是自身表现为inline-level box,内部表现为block-level box的元素。

那么inline-level 与 block-level 有何区别呢？

inline-level

  • 一个行内元素只占据它对应标签的边框所包含的空间，无法设置宽和高。
  • 默认情况下不会出现换行情况
  • 一般情况下，行内元素只能包含数据和其他行内元素

该类型元素强调的是行内内容，如文本，行内图片

block-level

  • 会换行，默认情况下不会同一行出现两个blocl-level 元素
  • 块级元素占据其父元素（容器）的整个空间，也就是宽度尽可能的宽
  • 一般块级元素可以包含行内元素和其他块级元素。

该类型元素强调的是段落，区块

inline-block VS inline

inline-block 元素

该类型元素在自身布局时，具有inline-level的特性：

  • 不换行，元元需其他元素在左边和右边

同时对内部而言具有block-level的特性：

  • 能够设置height和width
  • 具有margin和padding属性

inline 元素

  • 不换行
  • margin属性,仅有margin-left和margin-right生效
  • 无法设置height和width

补充

inline-block 元素的对齐方式是以其内部最后一个inline元素的基线为基准的，因此使用inline-block时经常会出现无法对齐的现象。

<style>
    #test{
        border: solid 1px yellow;
        font-size: 12px;
    }

    #test span{
        border: solid 1px green;
    }

    .inlineBlock{
        display: inline-block;
        width: 100px;
        height: 60px;
        border: solid 1px red;
    }
</style>

<div id='test'>
    <div class='inlineBlock'>
    </div><div class='inlineBlock'>
        <span>first element</span></br>
        <span>last element</span>
    </div>
    <div class='inlineBlock'></div>
</div>

解决方式

为修改inline-block 元素的 vertical-align 属性为 top

<style>
    .inlineBlock2{
        vertical-align: top;
        display: inline-block;
        width: 100px;
        height: 60px;
        border: solid 1px red;        
    }
</style>
